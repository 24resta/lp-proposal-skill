# 提案書pptx生成ルール v2

## 目的
分析結果をもとに、お客様向けLP制作提案書をpptxで自動生成する。
v2ではユーザーのカスタム設定（枚数・密度・クライアント特性）を反映する。

## 技術要件
- pptxgenjs を使用（`npm install -g pptxgenjs`）
- アイコンは react-icons + sharp でPNG化
- 出力先：`proposal_draft.pptx`（初稿）/ `proposal_final.pptx`（最終版）

## カスタム設定の反映方法

### スライド枚数
SKILL.mdのPhase 0で確定した枚数に従う。

**5枚（エグゼクティブサマリー）:**
```
1. 表紙
2. 競合分析サマリー＋差別化戦略（1枚に凝縮）
3. LP構成案（全体図）
4. FVイメージ
5. スケジュール・見積もり
```

**10枚（標準）:**
```
1. 表紙 → 2. 目次 → 3. 市場環境 → 4. 競合比較表
→ 5. 差別化戦略 → 6. ペルソナ → 7. LP構成案（戦略意図付き）
→ 8. FVイメージ → 9. スケジュール・見積もり → 10. クロージング
```

**15枚（詳細）:**
標準10枚に以下を追加：
```
4-b. 競合A社 個別分析
4-c. 競合B社 個別分析
7-b. CTA戦略の詳細
8-b. セクション別デザイン方針
11. SEO/広告連携の提案
```

**20枚（フル提案）:**
15枚に以下を追加：
```
3-b. 業界トレンド詳細
9-b. A/Bテスト計画
9-c. KPI設計
12. 運用・改善サイクル
13. 制作体制・メンバー紹介
```

**ユーザー指定枚数（例：8枚、12枚など）:**
最も近い標準パターンをベースに、スライドを追加 or 削除して調整する。

### 情報密度

**ライト（1スライド1メッセージ）:**
- タイトル + 1つの大きなビジュアル or 数字
- テキストは最小限（キャッチコピーレベル）
- 余白を贅沢に使う
- フォントサイズ：タイトル44pt、本文18pt
- 1スライドの要素数：最大3つ

**標準:**
- タイトル + 2〜3ポイント + 補足テキスト
- テーブル、カードは使うが詰め込まない
- フォントサイズ：タイトル36pt、本文14pt
- 1スライドの要素数：最大6つ

**リッチ（詳細データ込み）:**
- データテーブル、グラフ、詳細な数字
- 注釈や出典を明記
- フォントサイズ：タイトル28pt、本文12pt
- 1スライドの要素数：制限なし

### クライアント特性

**シンプル好き:**
- ライト密度に近づける
- ビジュアル重視、テキスト最小
- アニメーション指示を付ける（「このスライドは口頭で補足」等のノート）
- スピーカーノートに補足情報を入れる

**詳細好き:**
- リッチ密度に近づける
- データ、数字、根拠を多めに
- 比較表、グラフを積極的に使用
- 出典を明記

**不明:**
- 標準密度で作成
- 補足情報はスピーカーノートに入れておく

### トーンによるカラー調整
| トーン | 変更点 |
|---|---|
| 信頼感重視 | デフォルト（Ocean Gradient）のまま |
| 先進性重視 | navyをダーク（"0D1117"）、accentをパープル（"7C3AED"）に変更 |
| 親しみやすさ重視 | navyをティール（"0F766E"）、accentをオレンジ（"F97316"）に変更 |

## 戦略メモボックス（v1と同じ）
省略せず、必ず配置する。提案書の差別化要素。

## 松竹梅パッケージ
「あり」の場合のみ生成。金額は `¥XXX,XXX` のプレースホルダー。
```
梅：AI丸投げ型 / LP1枚 / 最短2週間
竹：ハイブリッド型 / LP1枚 / 3〜4週間（おすすめ）
松：プロ品質型 / LP+下層 / 4〜6週間
```

## 目次カスタム対応
ユーザーが目次を指定した場合、指定された構成に従ってスライドを生成する。

例：
```
ユーザー指定：
1. 表紙
2. 私たちについて
3. 市場分析
4. 競合比較
5. ご提案するLP
6. デザインイメージ
7. 進め方
8. お見積もり

→ この8枚構成に従って生成。分析結果を適切なスライドに割り当てる。
```

## ブラッシュアップ対応（Phase 3-B）

### アップロードされたpptxの処理
```bash
# テキスト抽出
python -m markitdown uploaded_proposal.pptx

# 前回の生成内容と差分をチェック
# → 変更点を報告
# → 改善提案を行う
# → ユーザーの指示に従って再生成
```

### 部分修正の対応
ユーザーが「3枚目だけ直して」等の指示をした場合：
1. 既存pptxをunpack
2. 該当スライドのXMLのみ編集
3. 再pack

全体再生成より部分修正を優先する（ユーザーの手直しを壊さないため）。

## 避けるべきこと（v1と同じ + 追加）
- テキストだけのスライド
- 全スライド同じレイアウト
- タイトル下のアクセントライン
- ユーザーが手直しした内容を勝手に戻す（最重要）
- カスタム設定を無視してデフォルトで生成する
