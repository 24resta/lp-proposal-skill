## 🤔 このツールは何？

LP（ランディングページ）の提案資料を作るとき、こんな作業をしていませんか？

- 競合サイトを1つずつ開いて、スクショを撮って、Excelにまとめる
- 差別化ポイントを考えて、構成案をパワポに起こす
- ワイヤーフレームをFigmaで手作業で引く

**このツールは、上記のすべてをAIが自動でやってくれます。**

あなたがやることは「**業界名を入力する**」だけ。あとはAIが対話しながら、以下を順番に生成します：

```
あなた：「LP提案を作りたい。業界：クラウド勤怠管理SaaS」

     ↓ AIが自動で実行 ↓

✅ 競合5社の調査・比較表
✅ 差別化戦略・ペルソナ設計
✅ LP構成案（セクションごとの戦略意図付き）
✅ 提案書パワーポイント（.pptx）
✅ Figmaワイヤーフレーム（実寸サイズ）
```

各ステップで「ここ変えたい」「もう少しデータ入れて」と壁打ちもできるので、AIが勝手に暴走することはありません。

## 💡 なぜこのツールが必要なのか

### Web制作会社の「提案フェーズ」の課題

LP制作の案件では、実際のデザイン・コーディングに入る前に「提案資料」が必要です。この提案フェーズには、通常こんな課題があります：

**① 時間がかかりすぎる**

競合調査だけで半日〜1日。構成案・提案書の作成でさらに1日。ワイヤーフレームを引くとさらに半日。合計で2〜3日かかることも珍しくありません。

**② 属人化している**

「競合のどこを見るか」「どう差別化するか」のノウハウが個人の頭の中にあり、チームで共有できていません。担当者が変わると提案の質が落ちます。

**③ 壁打ち相手がいない**

「この構成で本当にいいのか？」「このコピーで刺さるのか？」と悩んでも、すぐに壁打ちできる相手がいません。

### このツールが解決すること

| 課題 | このツールでの解決策 |
|------|---------------------|
| 時間がかかる | **2〜3日 → 約30分**に短縮 |
| 属人化 | ノウハウがSKILLファイルに蓄積され、**誰が使っても同じ品質** |
| 壁打ち相手がいない | 各ステップでAIが**選択肢を提示して対話** |
| 提案の質にバラつき | **競合5社・5軸分析**が毎回標準で入る |
| ワイヤーが後回しになる | 提案書と同時に**Figmaワイヤーも自動生成** |

### 競合ツールとの違い

「AIで提案書を作る」ツールは他にもありますが、このSKILLの特長は以下です：

- **Web制作に特化**：汎用のAI提案ツールではなく、LP制作のフローに最適化
- **対話型**：一発生成ではなく、各フェーズで壁打ち・カスタマイズが可能
- **Figma連携**：提案書だけでなく、実寸ワイヤーフレームまで一気通貫で生成
- **育てられる**：案件を重ねるたびにSKILLファイルを改善でき、提案の質が上がっていく
- **社内ノウハウが蓄積**：成功事例が `examples/` フォルダに溜まっていく

## 🔧 何ができるのか（機能一覧）

| 機能 | 説明 | 詳細 |
|------|------|------|
| 🔍 **競合調査の自動化** | Web検索で業界の主要5社を自動で分析 | FV訴求・ターゲット・価格帯・差別化ポイント・CTAの5軸で比較表を作成。市場規模・成長率のデータも自動収集 |
| 🧠 **戦略設計の自動化** | 競合の「穴」を見つけて差別化戦略を策定 | 競合の弱点を4つ以上特定、差別化戦略3案、ペルソナ3パターン、FVキャッチコピー4案（メイン1＋代替3）を生成 |
| 📋 **LP構成案の自動設計** | セクションごとに戦略意図を付けた構成案 | 8〜10セクションの構成案を自動設計。各セクションに「なぜこの順番か」「何を狙っているか」の戦略メモ付き |
| 📊 **提案書pptxの自動生成** | すぐに使えるパワーポイント提案書 | 5枚 / 10枚 / 15枚 / 20枚の4パターンに対応。情報密度（ライト / 標準 / リッチ）も選べる |
| 🎨 **Figmaワイヤーフレーム** | Figma MCP連携で実寸ワイヤーを直接生成 | 幅1440pxの実寸サイズ。各セクションにラベル＋戦略メモを配置。そのままデザインに着手できる状態 |
| 💬 **対話型カスタマイズ** | 各ステップで壁打ち・修正が可能 | 「このセクション順番変えたい」「ペルソナの年齢層を変えたい」などの指示にリアルタイムで対応 |
| 🔄 **ブラッシュアップ** | 手直しした.pptxをアップロードして再調整 | 手動で直した提案書をClaude Codeに読み込ませ、追加の改善提案や再生成が可能 |

### Step 1：Node.js をインストールする

**なぜ必要？** このツールが提案書（.pptx）を自動生成するとき、内部で JavaScript のプログラムを動かしています。Node.js は、その JavaScript を実行するための土台です。

> 💬 Node.js って何？
> 
> 普段ブラウザ（ChromeやEdge）の中でしか動かない JavaScript を、パソコン上で直接動かせるようにするツールです。このSKILLでは、パワーポイントファイルの生成やアイコンの変換に使っています。「裏方で動いてくれるエンジン」くらいのイメージでOKです。

**インストール手順：**

1. ブラウザで **[https://nodejs.org](https://nodejs.org)** を開く
2. トップページに大きなボタンが2つ表示されます → **左側の「LTS」と書いてある方** をクリック
   - 「LTS」= Long Term Support（長期サポート版）の略。安定版です
   - 右側の「Current」は最新機能版ですが、不安定な場合があるので選ばないでください
3. ダウンロードされたファイル（`node-v○○.○○.○-x64.msi`）をダブルクリック
4. インストール画面が出てきたら、**すべて「Next」を押し続ける**
   - 途中で「Add to PATH」というチェックボックスが出たら、**チェックが入っていることを確認**（通常は最初から入っています）
   - 「Tools for Native Modules」のチェックボックスが出た場合は、**チェックを入れなくてもOK**（入れると追加のツールがインストールされますが、このSKILLでは不要です）
5. 最後に **「Install」** をクリック
6. 完了したら **「Finish」** をクリック

```
💡 ポイント：
   LTSのボタンさえ選べば、あとは「Next」連打で大丈夫です。
   途中の設定画面は全部デフォルトのままでOK。
```

**確認方法：**

スタートメニューの検索バーに「PowerShell」と入力して、**Windows PowerShell** を開きます。以下をコピー＆ペーストして Enter を押してください：

```powershell
node --version
```

`v24.○○.○` のように表示されれば成功です（数字は時期によって変わります）。

ついでに npm（パッケージ管理ツール、Node.js と一緒にインストールされます）も確認：

```powershell
npm --version
```

`11.○.○` のように表示されればOKです。

```
⚠️ うまくいかないとき：
   「node が認識されません」と表示された場合は、PowerShell を一度閉じて、
   新しいウィンドウで開き直してからもう一度試してください。
   インストール直後はPATH（パソコンが「nodeはここにあるよ」と覚える設定）が
   反映されていないことがあります。
```

### Step 2：Git for Windows をインストールする

**なぜ必要？** Claude Code が内部で Git というツールを使っているため、先にインストールしておく必要があります。Git はプログラムのバージョン管理ツールですが、ここでは「Claude Code を動かすために必要な部品」くらいの理解で大丈夫です。

1. ブラウザで **[https://git-scm.com](https://git-scm.com)** を開く
2. 画面右側の大きな **「Download for Windows」** ボタンをクリック
3. ダウンロードされたファイル（`Git-○○-64-bit.exe`）をダブルクリック
4. インストール画面が出てきたら、**すべて「Next」を押し続ける**（設定は変えなくてOK）
5. 最後に **「Install」** をクリック
6. 完了したら **「Finish」** をクリック

```
💡 ポイント：
   設定画面がたくさん出てきますが、全部デフォルトのままで大丈夫です。
   とにかく「Next」→「Next」→「Next」→「Install」→「Finish」です。
```

**確認方法：**

先ほど開いた PowerShell に以下をコピー＆ペーストして Enter を押してください：

```powershell
git --version
```

`git version 2.○○.○` のように表示されれば成功です。

**方法B（ターミナル版）：PowerShell からインストールする**

ターミナル操作に慣れている方、またはプロジェクトフォルダ内で直接使いたい方はこちら。

1. **Windows PowerShell** を開く（スタートメニューで「PowerShell」と検索）
2. 以下をコピー＆ペーストして Enter：

```powershell
irm https://claude.ai/install.ps1 | iex
```

3. インストールが完了したら、**PowerShellを一度閉じて、新しく開き直す**（これ重要！）
4. 以下を入力して Enter：

```powershell
claude
```

5. ブラウザが自動で開くので、Claudeのアカウントで **サインイン**

```
⚠️ よくあるミス：
   インストール後にPowerShellを開き直さないと「claude が認識されません」
   というエラーが出ます。必ず一度閉じて、新しいウィンドウで開いてください。
```

**「claude が認識されません」と表示された場合：**

PowerShellを開き直しても解決しない場合は、以下を実行してください：

```powershell
[Environment]::SetEnvironmentVariable("PATH", "$env:PATH;$env:USERPROFILE\.local\bin", [EnvironmentVariableTarget]::User)
```

その後、PowerShellを閉じて再度開き、`claude` と入力してください。

### Step 4：SKILL ファイルを配置する

Claude Code に「LP提案の作り方」を教えるファイル群を配置します。

1. このリポジトリをダウンロードする（GitHubの **「Code」→「Download ZIP」**）
2. ZIPファイルを展開する
3. 作業用のフォルダを作る（例：`C:\Users\あなたの名前\lp-proposal`）
4. 展開したファイルの中身を、以下のように配置する：

```
C:\Users\あなたの名前\lp-proposal\
└── .claude\
    └── skills\
        └── lp-proposal\
            ├── SKILL.md           ← メインの実行フロー
            ├── research.md        ← 競合調査のルール
            ├── analysis.md        ← 分析・戦略設計のルール
            ├── proposal-pptx.md   ← pptx生成のルール
            ├── wireframe-figma.md ← Figmaワイヤー生成のルール
            └── examples\          ← 成功事例を蓄積するフォルダ
```

**ターミナルでやる場合（方法Bの人向け）：**

```powershell
# 作業フォルダを作成
mkdir C:\Users\$env:USERNAME\lp-proposal
cd C:\Users\$env:USERNAME\lp-proposal

# GitHubからクローン（URLは実際のリポジトリに置き換えてください）
git clone https://github.com/YOUR_USERNAME/lp-proposal-skill.git temp
mkdir -p .claude\skills\lp-proposal
Copy-Item temp\* .claude\skills\lp-proposal\ -Recurse
Remove-Item temp -Recurse -Force

# Claude Code を起動
claude
```

### 実際の会話例

**例1：カスタム生成（おすすめ）**

```

AI：スライドの枚数は？（5 / 10 / 15 / 20枚）

あなた：15枚

AI：情報密度は？（ライト / 標準 / リッチ）

あなた：リッチ。クライアントがデータ好きなので。

AI：設定まとめ：
    ・業界：クラウド勤怠管理SaaS
    ・ターゲット：従業員10〜50名の中小企業
    ・クライアント：テスト株式会社
    ・スライド：15枚
    ・情報密度：リッチ
    Phase 1（競合調査）に進んでよいですか？

あなた：OK

AI：（競合5社を自動調査…）
    競合5社の調査結果です。
    ・KING OF TIME — シェア1位、価格300円/人

    ・freee人事労務 — 会計連携が差別化
    ・Touch On Time — 打刻デバイス豊富

    追加・変更したい企業はありますか？

あなた：いいね、これで進めて

AI：（分析・戦略設計…）
    ...

## ⚙️ カスタマイズできること

### スライド枚数

| 枚数 | 用途 | 内容 |
|------|------|------|
| **5枚** | エグゼクティブサマリー | 表紙 → 競合＋差別化 → LP構成案 → FVイメージ → 見積もり |
| **10枚** | 標準提案 | 上記 ＋ 目次・市場環境・ペルソナ・クロージング |
| **15枚** | 詳細提案 | 上記 ＋ 競合個別分析・CTA戦略・デザイン方針・SEO提案 |
| **20枚** | フル提案 | 上記 ＋ トレンド詳細・A/Bテスト・KPI・運用提案・体制紹介 |

### 情報密度

| 密度 | 1スライドあたり | 向いているクライアント |
|------|----------------|----------------------|
| **ライト** | 1メッセージ + 1ビジュアル。余白多め | 忙しい経営者、ビジュアル重視の方 |
| **標準** | 2〜3ポイント + 補足説明 | 一般的なBtoB企業 |
| **リッチ** | データテーブル + 詳細な解説 | 分析好き、エンジニア、データドリブンな方 |

### 提案書のトーン

| トーン | 説明 |
|--------|------|
| **信頼感** | 落ち着いた配色、実績強調、安心感のあるデザイン |
| **先進性** | モダンな配色、技術力アピール、未来志向 |
| **親しみやすさ** | 暖色系、やわらかいトーン、中小企業向け |

## 🔧 よくあるトラブルと対処法

### インストール時のトラブル

| 症状 | 原因 | 対処法 |
|------|------|--------|
| **「node が認識されません」** | PATHが通っていない | PowerShellを閉じて新しいウィンドウで開き直す |
| **↑でも解決しない（node）** | インストール時にPATHが追加されなかった | Node.js を再インストール（「Add to PATH」にチェック確認） |
| **「claude が認識されません」** | PowerShellを開き直していない | PowerShellを閉じて、新しいウィンドウで再度開く |
| **↑でも解決しない（claude）** | PATHが通っていない | 上記「Step 3」の PATH追加コマンドを実行 |
| **ブラウザでのログインが終わらない** | クッキーブロック系の拡張機能 | 別のブラウザ（Edge等）で試す |
| **Git がインストールされていません** | Step 2をスキップした | [git-scm.com](https://git-scm.com) からインストール |

### 使用時のトラブル

| 症状 | 原因 | 対処法 |
|------|------|--------|
| **AIが壁打ちせずに突っ走る** | SKILLファイルのバージョンが古い | SKILL.md が v2.1以上であることを確認 |
| **pptx が生成されない** | 依存パッケージ未インストール | Claude Code に「必要なパッケージをインストールして」と指示 |
| **Figma連携が動かない** | Figma MCPの設定が必要 | Figma MCPの接続設定を確認（Starterプランだとレート制限あり） |
| **画像がペーストできない** | WindowsのClaude CodeはCtrl+Vで画像が貼れない | **Alt+V** を使う、またはファイルをドラッグ＆ドロップ |

## 📈 SKILLの育て方（運用改善）

このSKILLは「使えば使うほど賢くなる」設計です。案件を重ねるたびに以下を更新してください：

### 案件完了後にやること

| やること | 更新先 | 効果 |
|---------|--------|------|
| うまくいった分析パターンを記録 | `analysis.md` に追記 | 次回から同じパターンを自動適用 |
| クライアントに刺さったコピーを記録 | `analysis.md` に追記 | コピーの精度が上がる |
| デザインの改善点を記録 | `proposal-pptx.md` / `wireframe-figma.md` に追記 | 提案書・ワイヤーの品質が上がる |
| 成功事例をまるごと保存 | `examples/` にフォルダごと保存 | 次回のお手本になる |

### examples/ の保存フォーマット

```
examples/
├── 勤怠管理SaaS/
│   ├── analysis-output.md   ← 分析結果のアウトプット

│   └── figma-url.txt        ← FigmaファイルのURL
├── 人材派遣SaaS/
│   └── ...

    └── ...

## 🛠️ 技術スタック

| 技術 | 用途 |
|------|------|
| **Claude Code** | AI実行環境。SKILLファイルを読み込んで対話型ワークフローを実行 |
| **pptxgenjs** | JavaScriptライブラリ。パワーポイントファイル（.pptx）の自動生成 |
| **react-icons + sharp** | アイコンの生成とPNG変換。提案書内のアイコン表示に使用 |
| **Figma MCP** | Figmaへの直接書き込み。ワイヤーフレームを自動生成 |
| **markitdown** | .pptxファイルのテキスト解析。ブラッシュアップ時の差分検出に使用 |
