# 分析・戦略設計テンプレート

## 目的
競合調査結果をもとに、クライアントのLP戦略を設計する。

## 実行手順

### Step 1：競合の「穴」を特定（最低4つ）
競合5社の比較表を見て、以下の観点で穴を探す：

- **価格の穴** → 全社横並びなら、価格で差別化できる余地はないか
- **訴求の穴** → 全社同じ切り口（例：「No.1」）なら、別の切り口で差別化
- **ターゲットの穴** → 特定セグメントに特化したLPが不在ではないか
- **機能の穴** → 新技術（AI等）を前面に出してるプレイヤーが少なくないか
- **CTAの穴** → 「資料DL」だけで、軽いCTA（診断、チャット等）がないか
- **コンテンツの穴** → 動画デモ、Before/Afterなど不足してる要素はないか

### Step 2：差別化戦略の策定（3つ）
穴の分析から、クライアントが攻めるべき戦略を3つ策定する。

フォーマット：
```
戦略1：{戦略名}
  - 根拠：競合の穴のどれに対応するか
  - 具体策：LPのどのセクションでどう実現するか

戦略2：{戦略名}
  - 根拠：...
  - 具体策：...

戦略3：{戦略名}
  - 根拠：...
  - 具体策：...
```

### Step 3：ペルソナ設計（3パターン）
ターゲット企業規模内で、異なる課題を持つ3人のペルソナを設計する。

各ペルソナに必要な要素：
- **名前**（ペルソナA/B/C）
- **属性**（職種 / 企業規模 / 業種）
- **課題**（具体的な困りごと。感情が伝わるレベルで）
- **刺さる訴求**（このペルソナに響くキャッチコピー的な一文）

設計のルール：
- 3人の課題は重複しないこと
- 全員がLPの構成全体に触れるが、特に刺さるセクションが異なること
- 訴求は「機能説明」ではなく「課題解決の未来像」で書く

### Step 4：LP構成案（8〜10セクション）
各セクションに以下を設定する：

| # | セクション名 | 目的 | 内容概要 | 戦略意図（なぜここに置くか） |
|---|---|---|---|---|
| 1 | FV | ... | ... | ... |
| 2 | 課題提起 | ... | ... | ... |
| ... | ... | ... | ... | ... |

FV構成の必須要素：
- **メインキャッチコピー** — 感情に刺す（機能説明ではなく課題起点）
- **サブコピー** — 具体的なメリット（数字を含む）
- **メインCTA** — 重いCV（トライアル、問い合わせ）
- **サブCTA** — 軽いCV（診断テスト、資料DL）
- **ビジュアル** — プロダクトUI or ダッシュボードのイメージ

### Step 5：FVキャッチコピー案
- メイン案：1つ（最も刺さると判断したもの）
- 代替案：3つ（異なる切り口で）

コピーのルール：
- 機能名を入れない（「AI自動集計で」ではなく「月末が3日早く終わる」）
- 質問形式 or 数字入りが効果的
- 競合が使っていない切り口を選ぶ

## 最終出力
以上を `analysis-output.md` として保存する。Phase 3（提案書生成）とPhase 4（ワイヤーフレーム生成）の入力として使用する。
