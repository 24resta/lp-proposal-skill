# 🚀 LP提案自動生成 SKILL for Claude Code

**業界名を入力するだけで、競合調査 → 施策提案 → 提案書 → ワイヤーフレームを自動生成する Claude Code 用 SKILL**

---

## デモ

```
あなた：「LP提案を作りたい。業界：クラウド勤怠管理SaaS」

AI：スタートメニューを表示 → ヒアリング → 競合5社調査
  → 差別化戦略 → LP構成案 → 提案書pptx → Figmaワイヤーフレーム
  全自動で生成（各ステップで壁打ち・カスタマイズ可能）
```

### 手動だと2〜3日 → SKILLなら30分

---

## 何ができるか

| 機能 | 説明 |
|------|------|
| 🔍 **競合調査の自動化** | Web検索で業界の主要5社を自動分析。FV訴求・価格・ターゲット・CTA等を比較表に |
| 🧠 **施策提案の自動化** | 競合の「穴」を特定し、差別化戦略・ペルソナ・LP構成案を自動設計 |
| 📊 **提案書pptxの自動生成** | 戦略メモ付きの提案スライドを自動生成。5/10/15/20枚対応 |
| 🎨 **Figmaワイヤーフレーム** | Figma MCPで実寸ワイヤーを直接生成。戦略メモ付き |
| 💬 **対話型カスタマイズ** | 各ステップで壁打ち・修正が可能。一発生成も選べる |
| 🔄 **ブラッシュアップ** | 手直しした pptx をアップロードして再調整 |

## クイックスタート

### 前提条件
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) インストール済み
- Node.js 18+
- Anthropic API キー設定済み

### セットアップ

```bash
# 1. リポジトリをクローン
git clone https://github.com/YOUR_USERNAME/lp-proposal-skill.git
cd lp-proposal-skill

# 2. 依存パッケージをインストール
npm install -g pptxgenjs react react-dom react-icons sharp
pip install "markitdown[pptx]" --break-system-packages

# 3. SKILLをClaude Codeプロジェクトに配置
mkdir -p your-project/.claude/skills/lp-proposal
cp -r .claude/skills/lp-proposal/* your-project/.claude/skills/lp-proposal/
```

### Figma連携（オプション）
ワイヤーフレーム自動生成を使う場合：
```json
// .claude/settings.json
{
  "mcpServers": {
    "figma": {
      "url": "https://mcp.figma.com/mcp"
    }
  }
}
```

## ワークフロー

```
Phase 0：ヒアリング
  ├ 業界名・ターゲット・クライアント名
  ├ スライド枚数（5/10/15/20枚）
  ├ 情報密度（ライト/標準/リッチ）
  ├ クライアント特性（シンプル好き/詳細好き）
  └ 目次カスタム（自由記述可）
  ↓ ⚡ 確認＆壁打ち
Phase 1：競合調査
  ├ 主要5社の自動特定
  ├ 5軸比較表
  └ 市場データ
  ↓ ⚡ 確認＆壁打ち
Phase 2：分析・戦略設計
  ├ 競合の「穴」特定
  ├ 差別化戦略・ペルソナ設計
  └ LP構成案＋FVコピー案
  ↓ ⚡ 確認＆壁打ち
Phase 3：提案書pptx生成
  ├ カスタム設定反映
  └ 戦略メモ付きスライド
  ↓ ⚡ 確認＆壁打ち（pptxアップロードで再調整可）
Phase 4：Figmaワイヤーフレーム
  ├ 実寸ワイヤー（1440px幅）
  └ 戦略メモ付き
  ↓
🎉 完了！
```

**⚡ = 壁打ちポイント** — 各Phaseで停止し、修正・調整ができます。

## カスタマイズ項目

### スライド枚数

| 枚数 | 用途 |
|------|------|
| 5枚 | エグゼクティブサマリー |
| 10枚 | 標準的なLP提案（デフォルト） |
| 15枚 | 詳細提案。競合個別分析含む |
| 20枚 | フル提案。A/Bテスト・KPI・運用まで |

### 情報密度

| 密度 | 特徴 | 向いてるクライアント |
|------|------|------|
| ライト | 1スライド1メッセージ | 忙しい経営者 |
| 標準 | 2〜3ポイント + 補足 | 一般的なBtoB |
| リッチ | テーブル・グラフ・詳細 | データ好き |

### クライアント特性

| タイプ | 対応 |
|------|------|
| シンプル好き | 余白多め、口頭補足前提 |
| 詳細好き | データ・数字・根拠多め |
| 不明 | 標準で生成 |

## ファイル構成

```
.claude/skills/lp-proposal/
├── SKILL.md              ← 全体フロー・メニュー・壁打ち制御
├── research.md           ← 競合調査テンプレート
├── analysis.md           ← 分析・戦略設計テンプレート
├── proposal-pptx.md      ← pptx生成ルール（枚数・密度対応）
├── wireframe-figma.md    ← Figmaワイヤー生成ルール
└── examples/             ← 成功事例を蓄積
```

---

## SKILLの育て方

案件を重ねるたびに改善 → SKILLが賢くなる。

1. **うまくいった分析** → `analysis.md` に追記
2. **刺さったコピー** → ルールに反映
3. **デザイン改善** → `proposal-pptx.md` を更新
4. **成功事例** → `examples/` に保存
